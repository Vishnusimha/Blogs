Q11 — Minimum Size Subarray ≥ Target (🧠 Medium Level)

Given target = 7

Hint 1 Variable sliding window (expand + shrink).
Hint 2 Update answer WHILE shrinking, not after.
Hint 3 How to calculate the min Length ? formula?.
Answer int currentLen = i - start + 1; While runningSum ≥ target: update minLen → shrink start.
Answer
  • Expand window → add arr[right]
  • When sum ≥ target → try shrinking from left
    • Update minimum length during shrinking
    • Continue until sum < target
    • Repeat


    int runningSum = 0;
    int start = 0;
    int minSubArrayLen = Integer.MAX_VALUE;

    for (int i = 0; i < arr.length; i++) {
        // Step 1: Expand window
        runningSum += arr[i];

        // Step 2: Shrink window while it is valid
        while (runningSum >= target) {
            // Step 3: Update minimum length BEFORE shrinking further
            int currentLen = i - start + 1; // Calculate current window size. why + 1 is because we are calculating the length of the sub array. and the index starts from 0. so we need to add 1 to get the correct length.
            minSubArrayLen = Math.min(minSubArrayLen, currentLen);

            // Step 4: Shrink from left
            runningSum -= arr[start];
            start++;
        }
    }

Q12 — Subarray Sum Equals K (Count) (Medium level Problem)

Use prefix sum + hashmap.

Hint 1 Check (prefixSum - k) in map.
Hint 2 Questioning! How do you store frequency of prefix sums ?
Hint 2 Questioning! On what condition we write if ?
Hint 2 Ans 

// subarray sum (j+1 → i) = prefixSum(i) - prefixSum(j)
// prefixSum(i) - prefixSum(j) = k
// prefixSum(j) = prefixSum(i) - k
// Have I ever seen a prefix sum equal to (current prefixSum - k)?
// If YES: Every such occurrence = one valid subarray ending at i
// So we check: freq.containsKey(prefixSum - k)
Hint 3 Questioning! How do we count ?
Hint 3 Ans! count = count + freq.get(prefixSum - k);
Answer Store frequency of prefix sums and accumulate count.
Answer CODE 

public static int subarraySum(int[] arr, int k) {
    int prefixSum = 0;
    int count = 0;

    HashMap<Integer, Integer> freq = new HashMap<>();
    freq.put(0, 1); // Base case: prefix sum 0 appears once

    for (int i = 0; i < arr.length; i++) {
        prefixSum += arr[i]; // Update prefix sum

        // subarray sum (j+1 → i) = prefixSum(i) - prefixSum(j)
        // prefixSum(i) - prefixSum(j) = k
        // prefixSum(j) = prefixSum(i) - k
        // Have I ever seen a prefix sum equal to (current prefixSum - k)?
        // If YES: Every such occurrence = one valid subarray ending at i
        // So we check, freq.containsKey(prefixSum - k).

        if (freq.containsKey(prefixSum - k)) { // Check if there is a prefix sum that when subtracted from current prefix sum gives k
            count = count + freq.get(prefixSum - k); // If such a prefix sum exists, add its frequency to count
        }
        freq.put(prefixSum, freq.getOrDefault(prefixSum, 0) + 1); // Store current prefix sum frequency
    }

    return count;
}

Q20 — Two Sum (Array Pattern)

Find two numbers whose sum = target.

Hint 1 — Identify the Pattern This is NOT a sliding window problem. This is a lookup problem: "Have I seen the required pair before?"
Hint 2 — What should you calculate first? For each number, calculate:

complement = target - current number

Then ask: "Does this complement already exist in my past seen elements?"
Hint 3 — What should the HashMap/HashSet store? Store the CURRENT number (what you have already seen), NOT the complement.

Because future elements will search in past values.
Step-by-Step Thinking (Interview Brain Flow) Step 1: Traverse the array once (single loop)
Step 2: For each element, compute complement = target - num
Step 3: Check if complement exists in map/set (past seen values)
Step 4: If found → answer
Step 5: Else store current num for future lookup
⚠️ Mistakes You Actually Made (From Your Code) 1. Earlier you stored complement instead of num (logic reversed)
2. You treated HashMap as frequency map even though problem needs only existence
3. Initial confusion: inserting before checking (wrong order)
4. Pattern confusion: thinking like counting instead of lookup
5. Naming map as "freq" which misleads thinking (this is not a frequency problem)

Core correction:
✔ Check first → then store current number
✔ Store past elements, not future expectations
Answer (Optimal ajavapractice.leetcode.Solution) 

public static String twoSum(int[] arr, int target) {

    HashSet<Integer, Integer> seen = new HashSet<>();

    for (int num : arr) {
        int complement = target - num;

        if (seen.contains(complement)) {
            return "complement: " + complement + " num: " + num;
        }

        seen.add(num);
    }

    return "Not found";
}
